Section 02

Phased Deployment & Regulatory Strategy

Each phase maps to a distinct FDA regulatory basis — allowing earlier market entry while building toward full clinical product.

Phase 1 — Pilot

Display & Record

MDDS Enforcement Discretion
  • Thermal feed displayed on bedside screen — no analysis, no automated alerts
  • Raw zone temperatures written to EHR Flowsheet via Capsule → Health Connect
  • Camera hardware is already 510(k) cleared (product code LHQ)
  • Software classified as MDDS: transfer, store, display only
  • Clinician uses off-label (legal under practice of medicine doctrine)
  • IRB protocol for clinical validation study at Cohen Children's
  • Cohen Children's Epic go-live: May 30, 2026 (Wave 2). Health Connect routes to legacy EHR pre-cutover, then to Epic — no NeoTherm changes needed
EHR integration: Flowsheet rows (zone temps) via HL7v2 through existing Capsule + Health Connect. Health Connect handles EHR transition transparently — NeoTherm deploys once.
Phase 2 — Trending

CPTD Computation & Trending

Enforcement Discretion — Vital Sign Trending
  • Edge box computes CPTD (core − peripheral) and writes to Flowsheet
  • Historical trend lines visible in Epic Flowsheet graphing
  • No automated alerts from NeoTherm software
  • Northwell Epic team configures native BPA rule on CPTD threshold
  • FDA enforcement discretion covers "historical trending and comparison of vital signs"
  • FHIR R4 Observation resources written to Health Connect FHIR repository
Epic integration: Flowsheet + hospital-configured BPA. FHIR write-back via Health Connect. No custom Epic development.
Phase 3 — Cleared Product

AI-Powered Sepsis Early Warning

510(k) or De Novo Authorization
  • AI risk scoring, automated alerts, multi-condition detection (sepsis + NEC)
  • Predicates: Sepsis ImmunoScore (DEN230036), NAVOY CDS (510(k))
  • SMART on FHIR app in Epic App Market — dedicated NeoTherm widget
  • CDS Hooks integration: alert fires when clinician opens patient chart
  • PCCP (Predetermined Change Control Plan) for model updates post-clearance
  • QMSR compliance (21 CFR 820, effective Feb 2026)
Epic integration: Full — Flowsheet + BPA + SMART on FHIR widget + CDS Hooks via Health Connect FHIR APIs
Section 03

Integration Protocol Detail

Actual message formats flowing through each integration path — HL7v2 for Phase 1 device integration, FHIR R4 for Phase 2–3 modern APIs.

NeoTherm edge box emits standard HL7v2 ORU^R01 messages on the medical device VLAN. Capsule receives this identically to any bedside monitor. Phase 1 sends raw zone temperatures only (no computed CPTD, no alerts):

MSH|^~\&|NEOTHERM|NICU_BAY1A|CAPSULE|NORTHWELL|20260315120000||ORU^R01|MSG001|P|2.6 PID|1||MRN12345^^^NORTHWELL||GARCIA^MARIA OBR|1|||NEOTHERM_THERMAL|||20260315120000 OBX|1|NM|CORE_TEMP^Core Temperature||36.8|degC|36.0-37.5||||F OBX|2|NM|L_HAND^Left Hand Temp||33.1|degC|||||F OBX|3|NM|R_HAND^Right Hand Temp||33.6|degC|||||F OBX|4|NM|L_FOOT^Left Foot Temp||31.5|degC|||||F OBX|5|NM|R_FOOT^Right Foot Temp||32.1|degC|||||F // Phase 2 adds: CPTD computation + alert level segments

Capsule forwards to InterSystems Health Connect via TCP/MLLP. Health Connect transforms and routes to EHR Flowsheet. This path requires zero custom EHR development — Northwell's biomed team configures NeoTherm as a new device type in Capsule, which they do routinely for new monitors.

In Phase 2–3, the edge box (or NeoTherm server) posts FHIR R4 Observation resources directly to Health Connect's FHIR endpoint. Authenticated via SMART on FHIR backend OAuth2.

POST https://healthconnect.northwell.edu/fhir/r4/Observation Authorization: Bearer <SMART-on-FHIR-token> { "resourceType": "Observation", "status": "final", "category": [{"coding": [{ "system": "http://terminology.hl7.org/CodeSystem/observation-category", "code": "vital-signs" }]}], "code": { "coding": [{"system": "http://loinc.org", "code": "8329-5"}], "text": "Core-Peripheral Temperature Difference" }, "subject": {"reference": "Patient/EPIC-FHIR-ID"}, "effectiveDateTime": "2026-03-15T14:30:00-05:00", "valueQuantity": { "value": 3.2, "unit": "Cel", "system": "http://unitsofmeasure.org" }, "interpretation": [{"coding": [{ "code": "H", "display": "High" }]}], "component": [ {"code": {"text": "Core Temp"}, "valueQuantity": {"value": 36.8}}, {"code": {"text": "L Hand"}, "valueQuantity": {"value": 33.1}}, {"code": {"text": "R Hand"}, "valueQuantity": {"value": 33.6}}, {"code": {"text": "L Foot"}, "valueQuantity": {"value": 31.5}}, {"code": {"text": "R Foot"}, "valueQuantity": {"value": 32.1}} ] }

Health Connect stores the Observation in its FHIR repository, transforms to HL7v2 if Epic expects it on that interface, or passes through as FHIR to Epic's Interconnect server. The FHIR repository provides queryable history for the SMART on FHIR widget's trend charts.

InterSystems Health Connect is the universal translator at Northwell. It receives data in any format and routes to the right destination. Here is the routing logic:

// Health Connect Routing Configuration (conceptual) Source: NeoTherm Edge Box Protocol: HL7v2 ORU^R01 (Phase 1) | FHIR R4 POST (Phase 2-3) Transport: TCP/MLLP (via Capsule) | HTTPS (direct) VLAN: Medical Device Network (10.x.x.x) Transform: HL7v2 → FHIR: FHIR Façade — OBX segments → Observation.component[] FHIR → HL7v2: Reverse Façade — if target expects v2 Route 1: Epic Flowsheet Format: HL7v2 ORU → Epic Interconnect → Flowsheet rows Mapping: CPTD → custom Flowsheet row ID, zone temps → individual rows Frequency: q15min summaries from continuous stream Route 2: FHIR Repository (Phase 2+) Format: FHIR R4 Observation stored in InterSystems IRIS Purpose: Queryable history for SMART on FHIR widget Retention: Per Northwell data retention policy Route 3: Audit Log Every message logged. Supports FDA post-market surveillance (21 CFR Part 803) and IRB clinical trial requirements.

Key advantage: during Northwell's Epic migration (consolidating 30+ EHRs), Health Connect can route NeoTherm data to Epic at one site and to a legacy EHR at another — same input from the edge box, different output destinations.

Northwell Deployment Requirements

Contractual and compliance requirements for deploying at Cohen Children's Medical Center. Note: Cohen Children's (New Hyde Park, LI) is in Northwell's Epic Wave 2, going live May 30, 2026. Pilot planning should coordinate with EHR transition timeline.

Required
Business Associate Agreement
Thermal images = PHI. Non-negotiable. Must be signed before any patient data flows. 60-day breach reporting. Penalties up to $1.9M per violation category.
Required
IRB Protocol
Clinical validation study requires Feinstein Institutes IRB approval. Dual parental consent (neonates, Common Rule Subpart B). NY Public Health Law Article 24-A compliance.
Required
IT Security Assessment
SOC 2 Type II (or Type I for pilot). HECVAT questionnaire. Penetration test reports. Northwell may require HITRUST CSF certification for production.
Required
MSA / SaaS Agreement
License, SLAs (99.9% uptime for edge box), IP ownership, liability caps (fees paid in prior 12 months), consequential damages waiver.
Required
Data Use Agreement
If using pilot data to improve the model. Expert Determination de-identification method required (not Safe Harbor — small NICU populations have re-identification risk).
Required
Clinical Engineering Review
Northwell biomed team validates edge box meets IEC 60601-1 for patient vicinity. Camera mounting review. Capsule device type configuration.
Insurance
E&O / Technology E&O
$2–5M coverage. Required by Northwell for any vendor deploying clinical software.
Insurance
Cyber Liability
$2–5M coverage. Covers data breach, PHI exposure, ransomware incidents involving NeoTherm systems.
Insurance
Product Liability
Separate coverage. "Adjunctive use only" labeling is the primary liability shield — device does not replace clinical judgment.

Why this timeline works: The Epic Wave 2 go-live (May 30) actually creates a natural window. Hospitals pause new integrations during go-live preparation and stabilization. Use Q2–Q3 for partnership formalization, IRB, and procurement. By Q4, Epic is stable and the integration team has bandwidth for NeoTherm. The dual-champion model (clinical + innovation) ensures the project doesn't stall in either track.
